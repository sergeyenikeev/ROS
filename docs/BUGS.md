# Известные баги и ограничения PatrolBot

- [ТРЕБУЕТ ПРОВЕРКИ НА ЖЕЛЕЗЕ] Реальный LiDAR и моторный драйвер ещё не подключались; воспроизведение: отсутствует; workaround: использовать mock-профиль запуска; проверить: совместимость с конкретным оборудованием Raspberry Pi.
- [ТРЕБУЕТ ПРОВЕРКИ НА ЖЕЛЕЗЕ] Push в `git@github.com:sergeyenikeev/ROS.git` недоступен из текущей среды из-за отсутствия SSH-ключа; воспроизведение: `git ls-remote git@github.com:sergeyenikeev/ROS.git`; workaround: выполнить push вручную после настройки SSH; проверить: права доступа к репозиторию.
- [ИЗВЕСТНО] Профиль `patrol.launch.py` уже ожидает исполняемый файл mission manager, который будет добавлен следующим этапом; воспроизведение: запуск `ros2 launch patrolbot_bringup patrol.launch.py` до завершения реализации mission manager; workaround: использовать `mock_bringup`, `slam`, `localization` или `navigation` до появления узла патруля; проверить: успешность полного запуска патрулирования после добавления mission manager.
- [ТРЕБУЕТ ПРОВЕРКИ НА ЖЕЛЕЗЕ] `serial`-драйвер базы пока реализован как безопасная заготовка без реального обмена по последовательному порту; воспроизведение: запуск `patrolbot_base` с `driver_mode=serial`; workaround: использовать `driver_mode=mock` или доработать обмен под конкретный контроллер; проверить: протокол моторов, задержки и корректность обратной связи.
